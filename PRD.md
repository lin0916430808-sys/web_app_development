# PRD：個人記帳本

## 1. 專案概述
- **背景與動機**：現代人常有不知錢花去哪裡的困擾，需要一個簡單、好用且直覺的工具來記錄日常開銷與收入，進而達到財務管理與控制預算的目的。
- **目標用戶**：有記帳需求、希望掌握個人財務狀況，但不希望花費過多時間操作複雜軟體的一般大眾。
- **核心價值主張**：提供簡單直覺的操作流程，讓使用者快速記錄收支，並能清楚查看每月總花費、結餘與明細。

## 2. 功能需求

以下為系統的主要功能清單與使用者故事：

1. **新增收支紀錄**
   - **使用者故事**：作為一個使用者，我希望可以新增單筆消費或收入紀錄（包含日期、金額、類別、備註），以便精準掌握每一筆金錢流向。
2. **查看歷史明細**
   - **使用者故事**：作為一個使用者，我希望可以列表查看所有歷史收支明細，以便回顧與追蹤過去的消費狀況。
3. **計算當月收支**
   - **使用者故事**：作為一個使用者，我希望系統可以自動計算並顯示當月的總收入、總支出與結餘，以便了解當月預算是否超支。
4. **編輯或刪除紀錄**
   - **使用者故事**：作為一個使用者，我希望可以修改或刪除記錯的收支紀錄，以便維持帳本內容的正確性。
5. **收支類別分類**
   - **使用者故事**：作為一個使用者，我希望在記帳時可以選擇不同的收支類別（如餐飲、交通、娛樂等），以便日後可依類別檢視開銷比例。

## 3. 非功能需求
- **技術限制**：
  - 後端框架：Flask
  - 前端渲染：Jinja2 (HTML/CSS)
  - 資料庫：SQLite
- **效能與安全考量**：
  - 系統畫面載入與操作回應時間應在 1 秒內。
  - 基本的資料輸入驗證，防止不正確的資料格式存入資料庫。
  - 防範常見的 SQL Injection 攻擊，確保 SQLite 資料庫存取安全。

## 4. MVP 範圍 (Minimum Viable Product)

- **Must Have (必須要有)**：
  - 新增單筆收支紀錄（必填欄位：日期、金額、收支類型、類別）。
  - 查看歷史收支明細列表。
  - 統計並顯示當月的總支出、總收入及結餘。
- **Should Have (應該要有)**：
  - 編輯與刪除既有的收支紀錄。
  - 系統內建預設的常用收支類別（如：餐飲、交通、薪水、其他）。
- **Nice to Have (有會更好)**：
  - 讓使用者自訂收支類別。
  - 視覺化圖表（如圓餅圖）顯示當月各類別的消費比例。
